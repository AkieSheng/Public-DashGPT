To implement interactive effects in Vega-Lite views, use the `params` attribute and `transform` attribute similar to the example below. Height and width of each view in the Vega-Lite specification should be fixed at 200.

Consider each view in the VA system as a vertex in a (flow) graph. If some interaction in view A causes a change in view B, there should be an edge from A to B in the graph. This graph should satisfy the following properties:

- Each vertex in the graph has at least one outgoing or incoming edge;
- The overview view must be the starting point of the graph, i.e. its in-degree is 0.

The flow graph actually guides you which parameters of the previous views should be used in each view. A parameter should be used if and only if its from the parent or ancestor view of the current view. Here is an example to use two parameters from two ancestor views simultaneously:

"transform": [
    {
        "lookup": "user_id", // The primary key to lookup in the other_table_1
        "from": {
            "data": {"url": "http://localhost:8888/other_table_1.csv"},
            "key": "user_id", // The primary key in the table used in current view
            "fields": ["user_gender"] // Fields in other_table_1 used for filtering
        }
    },
    {
      "filter": {"param": "selected_user_gender"}
    },
    {
        "lookup": "user_id", // The primary key to lookup in the other_table_2
        "from": {
            "data": {"url": "http://localhost:8888/other_table_2.csv"},
            "key": "user_id", // The primary key in the table used in current view
            "fields": ["user_category", "user_score"] // Fields in other_table_2 used for filtering
        }
    },
    {
      "filter": {"param": "selected_user_demographics"}
    }
]

Be careful not to confuse `lookup`, `from.key` and `from.fields` when using parameters from other views. The former two represents primary key in other view and current view separately, while the latter represents fields from other views (that are to be added in current view).
